# Folio 自动化检查 MVP（第一轮）

> 本文只定义第一轮“可落地校验”，不宣称完整自动 merge / release / 派单引擎已经实现。

## 1. 本轮目标

先做门禁与提醒，再谈自动执行。

## 2. 推荐第一批检查

### PR 检查
- PR 标题非空，且建议符合约定前缀
- PR 描述包含：变更摘要 / 为什么改 / 如何验证 / 风险与兼容性
- PR 关联至少一个 issue
- 关键命令结果已填写（如 `composer test`）

### Issue 检查
- issue 具备基本类型
- issue 描述具备背景、目标、验收标准
- 若进入 `blocked`，必须补 blocker 信息

### 治理文件检查
- 状态机定义文件格式有效
- 文档目录存在，不被误删

## 3. 本轮不承诺内容

以下能力不在本轮承诺范围内：

- 自动 merge
- 自动 release
- 自动派单
- 自动改标签回写所有状态

## 4. 建议实现顺序

1. PR 模板固化
2. Issue 模板固化
3. GitHub Actions 做基础文本校验
4. 后续再接标签同步与状态机执行器

## 5. 与状态机的关系

自动化检查只负责“规则校验”和“状态门禁”，不越权替代人的判断。状态变更可先人工执行，后续再逐步自动化。
